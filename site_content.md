## Seção 2: Domínio de Negócio (OPME)

### Por que isso importa?
Trabalhar com OPME exige precisão. Você não está apenas codificando—está garantindo que pacientes recebam os materiais certos, que hospitais sejam pagos corretamente e que auditorias sejam bem-sucedidas.

### Áreas-Chave:
1. **Rastreabilidade** - Vinculação de itens de estoque a cirurgias específicas
2. **Regras de Faturamento** - O coração financeiro da distribuidora
3. **Glosas e Auditoria** - Evitar erros que impedem pagamento

**Dica de Ouro:** Entender a regra de negócio por trás de cada funcionalidade o torna um desenvolvedor muito mais valioso.

## Seção 3: Tecnologias Priorizadas

### 🔴 PRIORIDADE MÁXIMA: Ecossistema Spring e Java

**Por que focar aqui?**
- Você já está usando no trabalho
- É a base do sistema legado E da modernização
- Dominar bem garante autonomia em ambas as frentes

**Conceitos Essenciais:**
1. **Inversão de Controle (IoC)** - Como o Spring gerencia a vida dos objetos
2. **Injeção de Dependência (DI)** - Padrão idêntico no legado e no moderno
3. **Controllers** - Recebem requisições HTTP
4. **Services** - Contêm a lógica de negócio
5. **DTOs (Data Transfer Objects)** - Transferem dados entre camadas
6. **DAOs (Data Access Objects)** - Acessam o banco de dados

**Como Dominar:**
- Entenda como a informação viaja: Banco de Dados → DAO → Service → Controller → Frontend
- Estude o Spring legado para entender o fundamento
- Aplique em Spring Boot moderno para ver a evolução

**Recursos Recomendados:**
- Documentação oficial do Spring Framework
- Tutoriais sobre Spring Boot e REST APIs
- Projetos práticos: crie uma API REST simples

---

### 🟠 PRIORIDADE ALTA: JavaScript e React

**Por que focar aqui?**
- A empresa está modernizando o frontend
- React é o futuro do seu trabalho
- Você terá oportunidades de contribuir em novas features

**Conceitos Essenciais:**
1. **ES6+ Fundamentos** - Arrow Functions, Destructuring, Promises
2. **Componentes React** - Blocos reutilizáveis de UI
3. **Hooks** - useState, useEffect, useContext
4. **Estado e Props** - Como dados fluem entre componentes
5. **Consumo de APIs** - Integração com o backend via JSON

**Como Dominar:**
- Comece com JavaScript puro antes de decorar React
- Entenda que React é apenas JavaScript com esteroides
- Observe como o JSF/JSP são repetitivos e como React resolve com componentes
- Pratique criando componentes reutilizáveis

**Recursos Recomendados:**
- Documentação oficial do React
- Tutoriais sobre Hooks e Context API
- Projetos práticos: migre um layout JSF para React

---

### 🟡 PRIORIDADE MÉDIA: JasperReports e SQL

**Por que focar aqui?**
- Você usa diariamente no trabalho
- Dominar bem acelera suas tarefas atuais
- Deixa tempo para focar em Java/React

**Conceitos Essenciais:**
1. **SQL Puro** - Entenda a origem dos dados
2. **Otimização de Queries** - Relatórios rápidos
3. **Estrutura de Relatórios** - Grupos, campos, filtros
4. **Integração com Spring** - Como JasperReports consome dados

**Como Dominar:**
- Estude o suficiente para resolver suas demandas atuais
- Otimize as queries que alimentam os relatórios
- Torne-se o especialista da equipe em JasperReports

**Recursos Recomendados:**
- Documentação do JasperReports
- Tutoriais sobre SQL e otimização
- Análise de relatórios existentes na empresa

---

### 🟢 PRIORIDADE BAIXA: JSF e JSP

**Por que não focar aqui?**
- São tecnologias antigas
- Você precisa apenas manter o sistema legado
- Gastar energia aqui é perder tempo que poderia dedicar a Java/React moderno

**Conceitos Mínimos Necessários:**
1. **Ciclo de vida do JSF** - Apenas o essencial para manutenção
2. **JSP Basics** - Sintaxe para correções rápidas
3. **Integração com Spring antigo** - Entender o contexto

**Como Estudar:**
- Estude apenas quando surgir uma demanda específica
- Use a documentação como referência, não como material de estudo
- Aprenda com o código existente na empresa
